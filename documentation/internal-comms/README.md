# Internal Communications

Professional templates and guidelines for internal communications, announcements, and updates.

---

## Overview

This skill provides structured approaches for creating clear, professional internal communications including:
- Announcements
- Updates and changelogs
- Memos and notices
- Process changes
- System updates

---

## When to Use

### ✅ Use for:
- Documenting system changes
- Announcing new features
- Updating processes
- Creating changelogs
- Writing README updates
- Team notifications

### ❌ Don't use for:
- External communications
- Marketing materials
- Customer-facing docs
- Personal notes

---

## Communication Types

### 1. Announcements

**Purpose:** Inform about new features, tools, or changes

**Structure:**
```markdown
# [Feature/Tool Name] Now Available

## What's New
Brief description of what's been added

## Why It Matters
How this helps users

## How to Use
Quick start guide or link to documentation

## Questions?
Where to get help
```

**Example:**
```markdown
# Document Processing Skills Now Available

## What's New
Added DOCX, PPTX, PDF, and XLSX processing skills to the Skills folder.

## Why It Matters
Create, edit, and analyze office documents programmatically.

## How to Use
See G:\My Drive\06_Skills\documentation\document-processing\

## Questions?
Check the README.md in each skill folder.
```

### 2. Changelogs

**Purpose:** Track changes over time

**Structure:**
```markdown
## [Version/Date] - YYYY-MM-DD

### Added
- New feature 1
- New feature 2

### Changed
- Modified behavior 1
- Updated process 2

### Fixed
- Bug fix 1
- Issue resolution 2

### Removed
- Deprecated feature 1
```

---

### 3. Process Updates

**Purpose:** Document changes to workflows or procedures

**Structure:**
```markdown
# Process Update: [Process Name]

## What Changed
Clear description of the change

## Why
Reason for the change

## Impact
Who/what is affected

## Action Required
What users need to do (if anything)

## Effective Date
When this takes effect

## Resources
Links to updated documentation
```

---

### 4. System Updates

**Purpose:** Notify about system changes, maintenance, or issues

**Structure:**
```markdown
# System Update: [System Name]

## Status
Current status (e.g., Updated, Maintenance, Issue)

## Details
What happened or will happen

## Timeline
When it occurred/will occur

## Impact
What users might experience

## Next Steps
What's being done or what users should do
```

## Best Practices

### 1. Be Clear and Concise
- Use simple language
- Get to the point quickly
- Avoid jargon when possible

### 2. Structure Consistently
- Use templates
- Follow established formats
- Make it scannable

### 3. Provide Context
- Explain why, not just what
- Include relevant background
- Link to related docs

### 4. Make It Actionable
- Clear next steps
- Specific deadlines
- Contact information

### 5. Keep It Updated
- Maintain changelogs
- Update documentation
- Archive old announcements

## Templates

### Quick Announcement Template

```markdown
# [Title]

**What:** [Brief description]
**Why:** [Reason/benefit]
**When:** [Date/timeline]
**How:** [Link to docs or instructions]
```

### Changelog Entry Template

```markdown
## YYYY-MM-DD

### Added
- [New items]

### Changed
- [Modified items]

### Fixed
- [Resolved issues]
```

### Update Notice Template

```markdown
# Update: [Topic]

**Status:** [Current state]
**Impact:** [Who/what affected]
**Action:** [What to do]
**More Info:** [Link to details]
```
